Why TerraCllaim

Built for production, not demos

Every feature is designed around real-world constraints — sites with millions of rows, shared hosting CPU limits, and admins who can't afford surprises.

🔍
Dry-Run Preview First
Every cleanup operation shows you exactly what it will touch — row counts, post titles, dates, and file paths — before a single byte is deleted.
⚡
Chunked Processing
A three-phase engine (start → chunk → finish) processes large datasets in small batches. No bulk DELETEs that lock tables. No PHP timeouts. No site downtime.
🍒
Cron Congestion Detection
Visualise every scheduled WordPress cron job on a 24-hour timeline. Red bands flag Cron Congestion — when 3 or more jobs fire in the same 5-minute window.
🎲
Toggle Controls
Each cleanup category has an independent on/off toggle. Green = included, grey = skipped. Categories you never want touched can be permanently disabled.
📷
Image Optimisation
Resize and recompress JPEG and PNG images exceeding configurable max dimensions or quality thresholds. Bulk convert PNGs to JPEG with preview before commit.
🔒
Zero Data Exfiltration
No telemetry. No analytics. No external API calls of any kind. All processing runs entirely within your WordPress installation. Your data stays on your server.
How it works

Four steps from install to clean

No configuration wizard. No account creation. No payment details.

1
Install
Download the zip, upload via Plugins → Add New, activate. Done.
2
Configure
Toggle which categories you want to clean. Set age thresholds. Takes 30 seconds.
3
Dry Run
See exactly what will be removed — counts, IDs, titles — before committing to anything.
4
Clean
Confirm and watch the chunked engine process your site safely, batch by batch.
